# 🚗 Insurance Claims Analytics – End-to-End Data Analytics Project

Insurance Claims Analytics is an **end-to-end data analytics and business intelligence project** focused on analyzing insurance portfolio performance using simulated policy and claims data.

The project simulates a **real-world insurance analytics workflow** using Python, SQL Server, and Power BI to transform raw insurance data into actionable business insights.



# 🧠 Project Overview

This project demonstrates how analytics teams analyze insurance portfolio performance using business metrics and dashboards.

The workflow includes:

- Synthetic data generation  
- Data validation using SQL  
- Data modeling in Power BI  
- KPI and insurance metric calculations  
- Portfolio risk analysis  

The objective is to **evaluate portfolio profitability and identify claim risk patterns across policy tenures.**



# 🎯 Objectives

- Analyze insurance portfolio performance  
- Measure premium collection vs claim payouts  
- Calculate **Loss Ratio and Claim Rate**  
- Identify risk patterns across policy tenures  
- Analyze claim trends over time  
- Build an interactive insurance analytics dashboard  


# 🛠️ Tech Stack

- **Python** – Synthetic dataset generation  
- **Pandas & NumPy** – Data processing  
- **SQL Server (SSMS)** – Data storage & validation  
- **SQL** – Data analysis queries  
- **Power BI** – Interactive dashboard development  
- **DAX** – KPI calculations  
- **Git & GitHub** – Version control  



# 📊 Dataset Details

Two datasets were generated for the project.

## Policy Sales Table

Contains policy purchase information.

Features include:

- Customer_ID  
- Vehicle_ID  
- Vehicle_Value  
- Premium  
- Policy_Purchase_Date  
- Policy_Start_Date  
- Policy_End_Date  
- Policy_Tenure  

Each row represents a **policy purchased by a customer**.



## Claims Table

Contains insurance claims made by customers.

Features include:

- Claim_ID  
- Customer_ID  
- Vehicle_ID  
- Claim_Amount  
- Claim_Date  
- Claim_Type  

Each row represents a **claim filed against a policy**.



# 🔄 Data Pipeline Architecture

1. Generate synthetic insurance dataset using **Python**
2. Export generated datasets into **CSV files**
3. Import datasets into **SQL Server**
4. Validate and clean data using **SQL queries**
5. Create relationships between tables in **Power BI**
6. Build KPIs and metrics using **DAX**
7. Develop interactive dashboards for insurance analytics

# 📈 Key Insights

✔️ The insurance portfolio shows a **206.75% loss ratio**, meaning claims exceed premiums collected.

✔️ **1-year policies have the highest loss ratio**, indicating higher financial risk.

✔️ **3-year policies show the best portfolio performance** with the lowest loss ratio.

✔️ Policies with longer tenure tend to show **higher claim frequency**.

✔️ Policy sales remain relatively **stable across months**, indicating steady portfolio growth.
